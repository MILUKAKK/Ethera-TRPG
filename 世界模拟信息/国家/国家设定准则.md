# 国家设定准则

> 此文件为新建或扩写国家档案的强制规范。所有国家设定必须遵守此准则。
> 样板参考：`地区大国_艾尔德兰王国/`（完整12文件实现）。

---

## 一、核心原则

### 1.1 每个国家都是一个系统

国家不是属性列表。国家是一个互锁的系统——政治、经济、军事、法律、社会、文化互相约束。一个维度的改变会传导到其他所有维度。

**禁止的做法**：写"军队：50,000人"而不解释这些人从哪来、谁付钱、谁批准、谁指挥、如果全部动员农业会怎样。

**要求的做法**：每个数字从上一个维度的数据推导而来。军队人数 = 人口×经济×财政×政治×行政的交叉约束。而不仅仅是写一个炫目的数字。

### 1.2 具体优于抽象

"国王很富有" ← 无用。
"国王年度收入约100万金，来源为王室直领地15%全国可耕地+关税" ← 有用。

"贵族抵制改革" ← 无用。
"十二公爵中5位反对、4位支持、3位摇摆。反对派领袖黑木公爵控制约18万人口，他反对土地税的核心原因是他的封地农业产出在过去十年间一直在下降" ← 有用。

### 1.3 冲突是引擎

一个运行完美的国家不需要玩家。设定中必须包含张力——财政死锁、继承焦虑、阶级冲突、军事劣势、外交孤立。这些张力是LLM推演剧情时的燃料。

## 二、12维文件体系

### 2.1 完整文件清单（按建设顺序）

| # | 文件 | 核心问题 | 依赖 |
|---|------|---------|------|
| 1 | `地图.md` | 它在哪？地形/河流/山脉/海岸/资源分布？ | 无 |
| 2 | `人口与城镇.md` | 多少人？住在哪？什么阶层？ | 地图 |
| 3 | `经济.md` | 种什么？产什么？贸易什么？ | 地图+人口 |
| 4 | `社会结构与阶级.md` | 谁是贵族？谁是农民？阶级间能否流动？ | 经济+人口 |
| 5 | `贵族与皇室.md` | 具体谁在统治？政治派系？继承状态？ | 社会结构 |
| 6 | `宗教.md` | 教会的政治角色？土地？武装？金融？ | 贵族+社会结构 |
| 7 | `财政.md` | 政府怎么收钱？怎么花钱？国债？预算审批？ | 经济+贵族+人口 |
| 8 | `行政结构.md` | 国王的命令怎么传到村镇？谁修路？ | 贵族+财政 |
| 9 | `法律与司法.md` | 什么合法？什么犯罪？审判程序？执法者？ | 贵族+宗教+社会结构 |
| 10 | `军事.md` | 军队编制？指挥链？后勤？动员？战争潜力？ | 全部前面维度 |
| 11 | `外交政策.md` | 对每个邻国的策略？条约？情报？ | 全部前面维度（尤其是军事） |
| 12 | `传统.md` | 节日？故事？歌谣？习俗？地区认同？ | 全部前面维度（赋予文化内容） |
| 13 | `国家总览.md` | **最后写。** 全部维度的互锁视图。 | 全部前面维度 |

**军事和外交必须在所有内部维度完成之后才能写**——因为军事是内部力量的外部投射，外交是内政的延伸。反之则不成立——内部维度不需要知道军事的细节才能设定。

### 2.2 为什么财政是第七步

经济（第3步）是实体生产——GDP、产业、贸易。财政（第7步）是政府收支——税收、预算、国债。它们之间隔着社会结构、贵族、宗教——这三者决定了**谁拿走了生产剩余、谁有权力征税、谁可以拒绝纳税**。先写完这三者，才能写出真实的政府财政——而不是凭空编一个"年收入500万金"。

## 三、每个文件的最低内容标准

### 3.1 地图.md
- 国境范围（边界用什么标记——河流/山脉/界碑？）
- 核心地理要素的ASCII或文字描述（山脉、河流、平原、海岸）
- 首都位置 + 主要城镇位置
- 关键距离表（首都→边境、首都→主要城镇、城镇间）
- 自然资源分布（矿产、森林、渔场、牧场）

### 3.2 人口与城镇.md
- 总人口
- 人口结构：年龄分布、阶层分布、城乡比例
- 各区域的人口分布 + 人口密度 + 区域经济特征
- 城镇层级体系（首都→区域中心→小城→村镇——每个有名字有规模有职能）
- 人力资源约束：识字率、适兵役人口、专业人才池

### 3.3 经济.md
- GDP估算 + 部门构成（农业/手工业/贸易/服务/矿业占比）
- 各产业的具体产出和产能瓶颈
- 关键资源的自给/依赖度（特别是铁料/盐/马匹）
- 区域经济差异（各区域人均收入、经济特征）
- 出口/进口结构 + 贸易伙伴 + 贸易依赖度
- 货币体系 + 区域物价差异 + 各职业工资水平
- **不含政府收支**（那是财政.md）

### 3.4 社会结构与阶级.md
- 阶级金字塔（至少6层，每层有具体人数、特权、生存状态）
- 阶级流动路径（上升和下降都要写）
- 行会制度（如果有——列出主要行会、成员、权力）
- 性别与家庭（继承、财产、婚姻、关键女性NPC的社会定位）
- 荣誉与耻辱（各阶层的荣誉观 + 决斗/羞辱的文化逻辑）
- 日常社交礼仪（阶层间互动、外乡人与担保制度）
- 社会断层线（至少3组结构性冲突：什么vs什么、为什么、张力程度）

### 3.5 贵族与皇室.md
- 皇室成员（姓名、年龄、性格、政治立场、相互关系）
- 全部主要贵族（列出姓名——不只是头衔。封地人口、政治派系、个人立场、可争取的条件）
- 议会/贵族会议制度（几票、投票规则、派系分布、通过改革需要什么条件）
- 贵族-国王的权力边界（国王能不经议会做什么？议会能阻止国王做什么？）
- 关键政治斗争（当前的——具体矛盾、各方立场、可能走向）
- 继承规则与当前的继承焦虑

### 3.6 宗教.md
- 该国的宗教体系（国教？多元？民间信仰？）
- 教会的组织结构（层级、人数、驻地）
- 教会的政治角色（与国王/贵族的关系——同盟？对手？中立？）
- 教会的经济力量（土地、收入来源、放贷）
- 教会武装（如有——规模、指挥权、军事价值）
- 关键神职者的个人政治立场
- 民间日常宗教生活（普通人怎么信？村镇的宗教场景？）
- 教会的政治杠杆清单

### 3.7 财政.md
- 政府收入结构（每种税的税率、征收方式、年收入、承担者）
- 征收效率与损耗
- 政府支出结构（每项支出的金额、刚性/弹性、审批方）
- 当前赤字/盈余状态 + 财政缓冲能力
- 国债（债权人、金额、利率、抵押、到期）
- 财政危机触发场景
- 关键财政官员（姓名+性格+政治立场）
- **如果存在税改——这是核心政治危机的财政根源**（谁受益、谁受损、票数计算）

### 3.8 行政结构.md
- 中央行政组织（国王的直接下属——谁管什么）
- 领主行政（以一个大领主为样本——列出行政人员、职能、月薪）
- 命令传递链（以一道具体命令为例——从国王签令到村镇执行需多少天、每个环节的摩擦成本）
- 公共工程（谁修什么——资金来源+审批+维护困境）
- 税收的物理运输（多少钱币/粮食、怎么运、损耗多少）
- 记录与档案（人口名单、土地记录——多不准确？）
- 行政对教会的依赖（教会维护哪些不可替代的记录？）
- 行政效率总评

### 3.9 法律与司法.md
- 司法体系（几套并行？各自的管辖范围+法律来源+审判者+上诉终点）
- 法律阶级性（不同阶层在不同法庭受审）
- 实体法律（法典基础+习惯法+教会法的交叉领域）
- 刑事犯罪分类与刑罚阶梯（至少三级罪行+至少八种刑罚——每种写明适用阶层）
- 至少三种罪行的具体判定（杀人/盗窃/叛国——不同阶层不同结果）
- 民事法律（财产/契约/婚姻/继承——具体条款和执行方式）
- 审判程序（不同法庭的审理流程：谁审、证据标准、是否有律师、是否有上诉）
- 执法（谁在执法？管辖范围？执法密度在城乡差异有多大？）
- 法律-阶级-剧情的交叉（主角可能触发的罪行清单+每种的最轻至最重结果）
- 当前政治的法律灰色地带

### 3.10 军事.md
- 指挥体系（树状图——从最高指挥官到小队长。指挥人员来源。指挥半径与通讯延迟）
- 编制体系（树状编制——小队→中队→大队→军团。常备军与征召兵的编制区别）
- 军费与财政（来源结构+支出分解+战争额外开支计算+能打多久）
- 军费审批流程（谁批准？政治瓶颈在哪？）
- 后勤与补给（全军每日消耗+补给线+战略粮仓+装备供应链+致命进口依赖）
- 封建动员体系（领主征召义务+实际短少原因+动员时间线+对农业的影响）
- 驻防与部署（和平部署地图+战时预案+防御漏洞）
- 战争潜力评估（对每个主要假想敌的兵力/后勤/财政/指挥对比+胜负判断）
- 单兵参考模板（每个兵种的HP/攻击/防御/属性/士气/特殊能力/装备/XP）
- **单兵模板放在最后——它依赖前面所有分析**

### 3.11 外交政策.md
- 外交机构（谁管外交？驻外使节名单+驻地+个人背景）
- 外交指令传递速度表（首都→各首都的天数）
- 战略目标（不超过三个——每个有手段+进展评估）
- 外交选项矩阵（对每个邻国：当前关系+可升级方向+升级障碍+降级风险）
- 对每个主要国家/势力的具体政策（至少一段——核心判断+当前政策+关键变量）
- 条约与国际义务（全部列出——现行效力评估）
- 外交-内政联动（内政僵局如何锁死外交选项）
- 情报（来源+可靠性+覆盖范围+当前最紧急的情报缺口）
- 危机升级阶梯（至少6级——每级的行为+对方可能回应+触发条件）

### 3.12 传统.md
- 全国性节日（每个节日的意义+怎么过+在村镇的具体场景）
- 地方性习俗（至少2-3个——只在特定区域或城镇）
- 民间故事与歌谣（至少3个——谁讲、什么时候讲、寓意是什么）
- 日常谚语（至少5条——融入NPC对话）
- 地区身份认同（不同区域对"我们是XX人"意味着什么）
- 生命仪式（出生/成年/结婚/死亡——每个的具体仪式和场景）
- 饮食传统（至少一张分阶层三餐表+一种标志性食物的做法）
- 身体语言（几个常用手势+含义）

### 3.13 国家总览.md — 最后写
- 基础数据表（一键导出：人口/领土/政体/财政收入/军费/国债/建国年份）
- 历史概要（简——详细的在各自文件里）
- **系统互锁视图**（ASCII或文字描述——展示各维度的因果关系链）
- 当前困境（最多四个——每个都是系统互锁的结果而非独立的"问题"）
- 国家力量评估（多维度评分表+潜力vs现状的对比）
- 与主要国家/势力的关系概要
- 剧情关联节点（国家层面的关键事件——主角可能以什么方式卷入）

## 四、建设顺序的强制约束

### 4.1 禁止跳步

每写一个文件之前，确认它所依赖的文件已经完成：
- 写人口之前 → 地图必须已存在
- 写经济之前 → 地图+人口必须已存在
- 写军事之前 → 前面全部9个文件必须已存在
- 写总览之前 → 前面全部12个文件必须已存在

### 4.2 发现前面文件有漏洞——返回修补

在写军事时发现"后勤依赖铁料——但经济文件没有说明铁料从哪来" → 回到经济文件补充铁料地理约束 → 再继续写军事。

这是正常的。建设不是一次性的——是迭代的。但每次迭代的修补方向是**向前的依赖文件**，不是向后。

### 4.3 数字一致性

同一个数字如果出现在两个文件中——必须一致。特别关键的数字约束：
- 总人口 → 出现在人口/经济/军事文件
- 财政收入 → 出现在经济/财政/军事文件
- 军费 → 出现在财政/军事文件
- 贵族封地人口 → 出现在贵族/人口/军事文件（征召基数）
- 教会土地/收入 → 出现在宗教/经济/财政文件

## 五、国家深度分级

并非每个国家都需要12文件完整设定。根据国家在剧情中的重要性分级：

| 深度等级 | 必需文件 | 适用 |
|---------|---------|------|
| **S级（样板级）** | 全部13个文件 | 主角所在国、主要敌对国。当前：艾尔德兰（12文件完成）。建议下一国：维斯特法尔帝国 |
| **A级（完整级）** | 5文件：总览+地图+人口+经济+军事（简）+外交（简） | 重要盟国/贸易伙伴/可能参战国。如：洛桑联邦、格鲁姆汗国 |
| **B级（概要级）** | 单文件：总览（包含简化的地图/人口/经济/军事段落） | 地区中等国家。如：铁冠王国、灰塔公国、赤砂王国 |
| **C级（条目级）** | 单文件——当前状态 | 小型/微型实体。如：霜语王国、白崖侯国、自由岛 |

**升级规则**：当剧情推进至某个B级或C级国家成为关键参与者时——按建设顺序将其升级至A级或S级。不能跳级——C→S会产生回头路。

## 六、LLM使用说明

当LLM需要推演一个国家的内部决策或外部行为时：

1. 确定该国在剧情中的深度等级
2. 如果该国尚未达到该等级——先按建设顺序补全必需文件
3. 从该国的国家总览出发——定位到具体维度的文件
4. 推理过程：内政优先于外交、财政优先于军事、经济优先于财政

**推理链示例**：帝国是否会发动第四次血河战争？
→ 帝国外交政策（目标×能力）
→ 帝国军事（兵力/后勤/动员时间/对艾尔德兰的兵力比）
→ 帝国财政（能支撑多久的战争）
→ 帝国贵族（继承问题是否会延迟或加速对外战争）
→ 帝国内政（奴隶制的不稳定是否迫使皇帝寻找外部敌人来团结内部）
→ 结论：不是"是或否"——是一组条件："如果X且Y且Z同时满足，则战争在T时间内爆发的概率为P%"
